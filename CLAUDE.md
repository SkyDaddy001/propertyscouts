## Documentation Standards

### Content Organization

Each project file follows a consistent structure:
1. **Project Identity** - Name, developer, brand, type, location, RERA details
2. **Project Overview** - Key positioning, design philosophy, highlights
3. **Location & Connectivity** - Address, distance to landmarks, transportation links
4. **Unit Types & Variants** - Floor sizes, configurations, orientation options
5. **Amenities & Facilities** - Clubhouse, wellness, recreation, infrastructure
6. **Growth Drivers** - Regional development projects that affect property value
7. **Social Infrastructure** - Nearby education, healthcare, retail
8. **Developer Profile** - Company background, track record, awards
9. **Compliance & Disclaimer** - Legal notes and disclaimers

### Update Guidelines

- **Fact Verification**: All project data (RERA numbers, dates, unit sizes, distances) must be extracted directly from official project documents or verified sales brochures
- **Specifications**: When specifications change, note that "subject to change" disclaimer applies
- **Metrics**: Keep measurements consistent (sq.ft. for areas, kilometers for distances)
- **Dates**: Use DD-MM-YYYY format for consistency (e.g., 25-06-2024)

### Tone & Style

- Fact-based, informative tone—avoid speculative language
- Use bullet points for scannability
- Maintain professional language appropriate for property marketing
- Include disclaimers where information is representational or approximate

## Common Tasks

### Adding a New Project

1. Create a new markdown file with project code name: `[ProjectName].md`
2. Include VERIFIED INFO EXTRACTION or AI KNOWLEDGEBASE in the title
3. Follow the standard 11-section structure above
4. Cross-reference new project in this file
5. Include all relevant RERA numbers and registration dates

### Updating Existing Project Information

1. Verify changes against latest project documents
2. Note the date of update within the file if significant changes occur
3. Keep historical information when specifications become "subject to change"
4. Always maintain compliance disclaimers

### Extracting Data for Sales/Marketing

- **RERA Compliance**: Reference RERA numbers and dates (these are binding legal registrations)
- **Accurate Distances**: Use documented distances from official brochures
- **Amenity Lists**: Use the comprehensive amenity sections which are regularly updated for marketing collateral
- **Developer Track Record**: Reference "40+ projects completed, 8.5 lakh sq.ft. delivered"
